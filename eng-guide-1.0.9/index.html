Roomenta is a web app designed and developed to facilitate the access management to a company's workspace, but it can be easily adapted to any other scenario.

Reach Roomenta

In order to use Roomenta you need to find the following link: Roomenta. The access can be done through the classic Sorint's login form as for all the other applications use.

This is how the main screen looks like:

Parts Related to User Interaction:

  • Top Bar
  • First column: Offices
  • Second column: Calendar
  • Third column: Schedules
  • Fourth column: My Reservations

Change Theme

At the left end of the Top Bar, you can find a switch button that allows you to change the theme of the application from dark to light.

Info

Next to the switch button there is another button which, when clicked, opens a pop-up message.

The pop-up message shows important information regarding the ability to access:

  • Roomenta Guide: the guide that you are actually reading now.
  • The Emergency Taskforce: by opening this link it is possible to get assistance regarding the use of the application (not technical bug). A blank e-mail is created, with Emergency Taskforce as sender.
  • Give Us Your Feedback: directly redirect to WeScrum, where it is possible to create a card in order to indicate to the Developers a bug or a change request.

Caution: Technical bug and assistance cover two different issues.

Example #1. Write an e-mail to Emergency Taskforce if you need assistance when booking a room.

Example #2. A confirmation button has a misaligned text and it doesn't seem to confirm any action: click Give Us Your Feedback!

Name and Versions

At the center is possible to see the name of the application, the version and whether it is currently in beta or not.

Username and Logout

On the right side you can see your own username and the logout button with which you exit the application, going back to login.

On the first column on the left you can read information regarding the head office and the room currently selected.

Selecting Head office and Room

You can expand the list of the rooms by using the relative tabs. Selecting a room automatically selects the head office to which it belongs. For every room are displayed:

  • Description: Usually the floor on which it is located inside the head office.
  • Capacity: The total number of the people that can occupy the room.
  • Monitor icon: Indicates whether or not there is a monitor inside the room based on whether or not the icon is activated.
  • Projector icon: Indicates whether or not there is a projector inside the room based on whether or not the icon is activated.

When a room is selected, the upper part of the column is updated with the information relative to the choice just made.

Favourites

Next to the name of each room there is a heart - shaped icon (ü§ç), through which is possible to add the selected room to your list of favourites. This latter is displayed as an expandable add-on tab just above the head offices tabs.

A room can be removed from the favourites list by clicking on the cross-shaped button present on the left side of room name.

To the right of each favourite room is shown an abbreviation of the head office where it is located in order to avoid misunderstandings between homonymous rooms.

In the foreground, in the second column, there are the current day and date.

Selecting the date

In this section there is a list of dates. By selecting a specific date, you can see its details. The date format is dd/mm/yyyy followed by the name of the day of the week.

Weeks are seperated from each other by a horizontal line. Saturday and Sunday are slightly darkened in order to give more relevance to working days.

When a date is selected, the upper part of the column is updated with the information relative to the choice just made.

In the third column you can read information relative to the currently selected room reservation limits. In particular the minimum and maximum duration of the reservation.

Selecting a Reservation Slot

In the central part of the column there is a table with three columns and seven rows whose cells include the start and end time of the slot.

Slots with a dark color are disactivated and can't be selected. These slots are being disactivated when they refer to time slots that can't be booked or to a time period that has passed.

It is possible to select the active slots by clicking on them. You notice the selection thanks to a change of color and outline thickness. In order to correctly complete a reservation, it is necessary to select a slot in a valid way.

Nota: The validity consists of selecting contiguous slots, i.e. not selecting a slot between two already selected. The validity is also given by respecting the constraints at the top of the column (already indicated above).

Caution: In case of violation of the selection validity, an information message will appear at the bottom left with the information necessary for correction. Until the selection is valid, it will not be possible to confirm the reservation.

Table Consultation

  • From the table you can also see the reservations. The reservations of other users differ from yours because of the different color.
  • Hovering the mouse over a reservation will highlight the slots it includes.
  • Clicking on a reservation opens a dialog contaning the details of it. The details consist of:
  • Description: a brief description added by the user at the time of the reservation
  • Guests list: list of the e-mails related to the guests of the reservation
  • Participants list: list of e-mails related to the guests that have confirm their presence through the confirmation e-mail.

Nota: When a reservation on the table is green it means that it is confirmed, in other words it has reached the minimun number of participants required in order not to be cancelled.

Switch Time Slots and Shifts

It has been introduced and made more visible a switch useful for those who work on shifts (e.g. service desk). We then move on to the display of the shifts that can be booked on the typical eight-hour shifts. The logic remains the same as before: you can, of course, book only future slots and not already booked.

Nota: This mode is visible only for reservations related to rooms with duration of maximum or equal to 8 hours. It doesn't appear for the rest. (If you want to try, select Sorint Hall room).

Invitations Section

Under the time slots table there is the part of the invitations. Here you can view the minimum number of invitations needed to complete the reservation and the maximum number of guests for the selected room.

You can add a person to be invited by inserting the name and a valid e-mail address in the appropriate fields (orange box), and clicking the button "‚úö" present on the right side of the box. Once the invitation is successfully completed, the name and the e-mail can be viewed in a list below.

It is also possible to delete the creator of the reservation from the guests list: in other terms you create reservations on behalf of other colleagues.

Reservation Description

Below the invitation section there is a text box in which you can specify a description for your reservation.

Caution: The description will be visible to all users once the reservation is completed.

If a description has already been specified and you want to change room or date, the description will be saved and then resumed when a valid slot selection is made.

Reservation Confirm

If the reservation is valid and respects all requirements the Confirm button is going to be activated, which can be disabled in case of any changes that invalidate the reservation.

Once the reservation is confirmed, a message is displayed at the bottom right that informs you about the result of the reservation. At this point a confirmation e-mail is sent to all the participants in the meeting.

Confirmation e-mail

When a user completes a reservation or is invited by another user, he/she receives an e-mail, to the address indicated by the person who has booked the room, containing the details of the reservation and the possibility to confirm his/her presence.

Nota: Once the minimum number of participants has been reached, the reservation is automatically confirmed.

Limit of Reservations

At the bottom of the column there is a message that indicates how many more reservations can be completed. If you wish to proceed with another reservation, despite you head reached the maximum number of reservations (per day and room), it is necessary to delete one of the reservations already made, in order to proceed with another one.

In the fourth column you can read information regarding the number of reservations completed.

Reservation card

There is a card for each reservation made that contains details of the same. You can see the date, the description, the start and end time and the room booked.

Nota: The end time of the reservation is indicated 15 minutes before the reserved time. This advance is necessary to comply with safety regulations due to the current health emergency situation.

The reservation not yet confirmed will have an orange color, while it will change to green once the minimum number of participants is reached.
